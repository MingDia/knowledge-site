# 🗃️ 卡片盒知识管理系统

> 基于卢曼卡片盒笔记法，构建个人知识库

---

## 📥 暂存箱（Inbox）
&gt; 临时记录随时产生的想法、灵感，需要定期整理到PARA分类

```dataview
LIST
FROM "999-Inbox"
SORT file.ctime DESC
LIMIT 10
```

## � 项目笔记（Projects）
&gt; 与当前进行中项目相关的笔记，有明确的完成目标和时间限制

```dataview
LIST
FROM "1-Projects"
SORT file.mtime DESC
LIMIT 10
```

## 🎯 领域笔记（Areas）
&gt; 属于长期关注领域的知识，需要持续维护和更新

```dataview
LIST
FROM "2-Areas"
SORT file.mtime DESC
LIMIT 10
```

## � 资源笔记（Resources）
&gt; 参考资料、知识库内容，没有明确时间限制，供随时查阅

```dataview
LIST

LIMIT 10
```

## 🗄️ 归档笔记（Archives）
&gt; 已完成的项目、不再活跃的内容，保留备查

## 💡 PARA + 卡片盒工作流
1. **收集阶段**：所有新想法先放入 `999-Inbox`，及时记录
2. **整理阶段**：定期整理Inbox，将笔记分类到对应PARA文件夹
   - 有明确完成目标和时间限制 → `1-Projects`
   - 长期关注领域，持续维护 → `2-Areas`
   - 参考资料、知识库内容 → `3-Resources`
   - 已完成或不再活跃 → `4-Archives`
3. **知识关联**：使用双链将相关笔记关联起来
4. **定期回顾**：每周检查Projects进度，每月回顾Areas更新
